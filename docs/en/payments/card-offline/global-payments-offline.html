1 Introduction

1.1 Document Overview

Physical card payment refers to the payment method where credit cards or debit cards are swiped on POS terminals for consumption. Today, credit cards are very common, and they can be used in supermarkets, electronics stores, shopping malls, restaurants, hospitals, online services, telecommunications providers, and many other places.

Merchants use the platform's API for integration. The merchant generates order parameters and provides them to the service provider to invoke payment. After successful payment, the merchant receives relevant payment information.

Scenario example:

Payment process:

Step (1): The user makes a purchase at a physical store, and the cashier brings out the POS machine.

Step (2): The user presents their credit card or debit card and swipes it on the POS machine for authorization.

Step (3): The user signs on the POS machine and the payment is deducted.

Step (4): The POS machine displays the payment result.

1.2 Target Audience

For reference by technical or business personnel of merchant platform service providers.

5 API Description

5.1 Create Order API

5.1.1 Business Function

After the merchant calls the Create Order API, the response parameters are generated and returned to the merchant. The merchant determines the payment channel to invoke the payment institution for payment operations. After payment completion, the payment institution calls back the platform, and the platform notifies the merchant API of the result.

5.1.2 Interaction Mode

Request: Backend synchronous request mode

Response: Backend synchronous request mode
