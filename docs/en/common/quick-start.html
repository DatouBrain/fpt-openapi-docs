Quick Start

Doc Version: V1.0

This guide walks you through integrating Fintech Payment OpenAPI from scratch, including environment preparation, signature computation, making payment requests, and handling payment notifications.

Before You Begin

Before starting, ensure you have the following:

ItemDescriptionHow to Obtain
Merchant AccountFintech Payment platform merchant accountProvided by platform business team
client_idClient ID, identifies merchantProvided via merchant portal or business team
store_idStore ID, identifies a specific storeProvided via merchant portal or business team
Merchant KeyCommunication key used for digital signaturesProvided by business team, keep it safe
Server EnvironmentServer supporting HTTPS POST requestsPrepared by merchant

Payment Flow

A typical payment flow:

📋 Business Flow

1. Create Order → Merchant server calls /pay to create an order
2. Invoke Payment → Merchant uses the returned payment link/QR code to guide the user to pay
3. Payment Notification → Platform asynchronously notifies the merchant's notify_url of the payment result
4. Order Query → Merchant actively calls /query to confirm order status
5. Refund (optional) → Merchant calls /refund to initiate a refund

All API calls must be made on the merchant server. Never call them directly from the client.

Quick Integration Steps

Step 1: Build Request Parameters

Assemble JSON request parameters, including business and common parameters:

JSON{
  "service": "pay.weixin.web.intl",
  "client_id": "1001",
  "store_id": "1001001",
  "order_no": "ORDER20260708001",
  "amount": 1.00,
  "body": "測試商品",
  "notify_url": "https://your-domain.com/notify",
  "callback_url": "https://your-domain.com/result",
  "client_ip": "127.0.0.1",
  "nonce_str": "abc123def456",
  "lang": "en_US"
}

Step 2: Compute Signature

Sort all parameters (except sign and lang) by ASCII code ascending, concatenate as QueryString, append the merchant key, then compute MD5:

PYTHONimport hashlib

params = {
    "service": "pay.weixin.web.intl",
    "client_id": "1001",
    "store_id": "1001001",
    "order_no": "ORDER20260708001",
    "amount": "1.00",
    "body": "測試商品",
    "notify_url": "https://your-domain.com/notify",
    "callback_url": "https://your-domain.com/result",
    "client_ip": "127.0.0.1",
    "nonce_str": "abc123def456",
}

# 1. Sort by ASCII code of key ascending
sorted_params = sorted(params.items())

# 2. Concatenate as key=value&key=value format (empty values not transmitted)
sign_str = "&".join(f"{k}={v}" for k, v in sorted_params if v)

# 3. Append merchant key
merchant_key = "YOUR_MERCHANT_KEY"
sign_str += f"&key={merchant_key}"

# 4. MD5 and convert to uppercase
sign = hashlib.md5(sign_str.encode("utf-8")).hexdigest().upper()
print(sign)

Step 3: Send API Request

PYTHONimport requests, json

params["sign"] = sign

url = "https://uat.fintechpaymenthub.com/openapi/v1/pay"
response = requests.post(url, json=params, headers={"Content-Type": "application/json"})
result = response.json()

if result.get("message") == "SUCCESS":
    print("Order created successfully!", result)
else:
    print("Order failed:", result)

Step 4: Handle Payment Notification

After payment, the platform sends a POST to notify_url with the result. Merchant should:

  • Verify the signature
  • Verify order no and amount match local system
  • Return the plain string success after processing business logic

Full Code Example

Below is a complete Python example covering order creation, query, and notification handling:

PYTHONimport hashlib, json, requests
from flask import Flask, request

app = Flask(__name__)
MERCHANT_KEY = "YOUR_MERCHANT_KEY"
BASE_URL = "https://uat.fintechpaymenthub.com/openapi/v1"

def sign(params, key=MERCHANT_KEY):
    sorted_params = sorted(params.items())
    sign_str = "&".join(f"{k}={v}" for k, v in sorted_params if v)
    sign_str += f"&key={key}"
    return hashlib.md5(sign_str.encode("utf-8")).hexdigest().upper()

# Create order
def create_order():
    params = {
        "service": "pay.weixin.web.intl",
        "client_id": "1001", "store_id": "1001001",
        "order_no": "ORDER20260708001", "amount": "1.00",
        "body": "測試商品", "notify_url": "https://your-domain.com/notify",
        "callback_url": "https://your-domain.com/result",
        "client_ip": "127.0.0.1", "nonce_str": "abc123def456",
    }
    params["sign"] = sign(params)
    resp = requests.post(f"{BASE_URL}/pay", json=params)
    return resp.json()

# Query order
def query_order(order_no):
    params = {
        "service": "pay.weixin.web.intl",
        "client_id": "1001", "store_id": "1001001",
        "order_no": order_no, "nonce_str": "query123456",
    }
    params["sign"] = sign(params)
    resp = requests.post(f"{BASE_URL}/query", json=params)
    return resp.json()

# Receive payment notification
@app.route("/notify", methods=["POST"])
def notify():
    data = request.get_json()
    # Verify signature
    recv_sign = data.pop("sign", "")
    if sign(data) != recv_sign:
        return "FAIL"
    # Verify order
    if data.get("message") == "SUCCESS":
        # Update local order status
        print(f"Order {data.get('order_no')} paid successfully")
    return "success"
