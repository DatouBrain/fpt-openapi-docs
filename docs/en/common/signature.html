Digital Signature

Doc Version: V1.1

To ensure authenticity and integrity during data transmission, all API requests and responses must be digitally signed. Signing is a two-step process: first assemble the raw signature string according to the rules, then compute the signature result using the MD5 algorithm.

Signature Raw String

For both requests and responses, the raw signature string is assembled as follows:

  1. Except for the sign and lang fields, all parameters are sorted by the ASCII code of the field name in ascending order and concatenated in QueryString format (i.e. key1=value1&key2=value2&...)
  2. Empty values are not transmitted and do not participate in signature string assembly
  3. Both field names and field values use their original values without URL Encode
  4. The platform's response or notification may add parameters due to upgrades; please allow for this when verifying the response signature
⚠️ IP Field Difference

QR Pay products use client_ip as the request parameter, while Fintech Payment Online card payment products use ip_address. When integrating Fintech Payment Online Card payment, replace client_ip in the examples below with ip_address.

Example

Assume the following request parameters:

JSON{
  "client_id": "1001",
  "store_id": "1001001",
  "service": "pay.weixin.web.intl",
  "order_no": "Me08772e914454656bf26ec8996e35bb",
  "amount": "1",
  "body": "a chair",
  "notify_url": "2031",
  "client_ip": "127.0.0.1",
  "nonce_str": "1409196838",
  "sign": "22C9835251DCF58BB70C78FA8D6448BD"
}

The correct sorted signature result (excluding sign and lang) is:

TEXTamount=1&body=a chair&client_id=1001&client_ip=127.0.0.1&nonce_str=1409196838&notify_url=2031&order_no=Me08772e914454656bf26ec8996e35bb&service=pay.weixin.web.intl&store_id=1001001

Signing with Nested Fields (e.g. items)

When a request contains JSON array or object fields such as items (e.g. the Create Order API for Fintech Payment Online card payment), the field still participates in the signature as a top-level field sorted by field name, and its value is represented as a compact JSON string with no extra whitespace (canonical JSON). The rules are as follows:

  1. Array elements keep their original order; object fields are sorted by name in ascending order
  2. null-valued fields are excluded from the object JSON string
  3. String values are wrapped in double quotes with special characters escaped
  4. items.quantity and items.unit_price are string fields and treated as strings (wrapped in quotes)
  5. No extra whitespace in the JSON string (no spaces, no line breaks)

Example, assume the following request parameters containing items:

JSON{
  "client_id": "1001",
  "store_id": "1001001",
  "service": "pay.master.sinopay.online",
  "order_no": "Me08772e914454656bf26ec8996e35bb",
  "amount": "246",
  "body": "Sinopay items test",
  "notify_url": "2031",
  "ip_address": "127.0.0.1",
  "nonce_str": "1409196838",
  "items": [
    {"description": "Mint (piece)", "quantity": "1", "unit_price": "246"}
  ]
}

The canonical JSON representation of items is:

TEXT[{"description":"Mint (piece)","quantity":"1","unit_price":"246"}]

The correct sorted signature result (excluding sign and lang) is:

TEXTamount=246&body=Sinopay items test&client_id=1001&ip_address=127.0.0.1&items=[{"description":"Mint (piece)","quantity":"1","unit_price":"246"}]&nonce_str=1409196838&notify_url=2031&order_no=Me08772e914454656bf26ec8996e35bb&service=pay.master.sinopay.online&store_id=1001001
💡 Backward Compatibility

When a request does not contain nested fields such as items (e.g. QR code payment, WAP, WEB, etc.), the new and old signature rules produce identical signatures, so existing developers need no changes. Only requests containing nested fields (such as the card payment items) need to follow the canonical JSON rules above.

Signature Algorithm

Use the MD5 digest algorithm, appending the merchant communication key to the raw signature string for computation, then convert the result to uppercase:

📐 Signature Formula

sign = MD5(raw signature string + "&key=" + merchant key).toUpperCase()

Note: The character encoding specified when converting the string to a byte stream must be UTF-8.

Full Example

Assume the merchant key is MjNiYWE4NGY2YTMxNDk2MTljODBkMjZkMjk0NjAxY2M=:

TEXTRaw signature string:
amount=1&body=a chair&client_id=1001&client_ip=127.0.0.1&nonce_str=1409196838&notify_url=2031&order_no=Me08772e914454656bf26ec8996e35bb&service=pay.weixin.web.intl&store_id=1001001

After appending the key:
amount=1&body=a chair&client_id=1001&client_ip=127.0.0.1&nonce_str=1409196838&notify_url=2031&order_no=Me08772e914454656bf26ec8996e35bb&service=pay.weixin.web.intl&store_id=1001001&key=MjNiYWE4NGY2YTMxNDk2MTljODBkMjZkMjk0NjAxY2M=

MD5 result in uppercase:
D89E5E3DF1447F61F6AC1EC4AB8B177E

Multi-language Code Examples

Python

PYTHONimport hashlib

def generate_sign(params, merchant_key):
    # Exclude sign and lang, sort by key ASCII code
    filtered = {k: v for k, v in params.items()
                if k not in ("sign", "lang") and v}
    sorted_params = sorted(filtered.items())
    # Concatenate as key=value&key=value
    sign_str = "&".join(f"{k}={v}" for k, v in sorted_params)
    # Append merchant key
    sign_str += f"&key={merchant_key}"
    # MD5 and convert to uppercase
    return hashlib.md5(sign_str.encode("utf-8")).hexdigest().upper()

Java

JAVAimport java.security.MessageDigest;
import java.util.*;

public String generateSign(Map<String, String> params, String merchantKey) throws Exception {
    // Exclude sign and lang
    params.remove("sign");
    params.remove("lang");
    // Sort by key
    List<String> keys = new ArrayList<>(params.keySet());
    Collections.sort(keys);
    // Concatenate
    StringBuilder sb = new StringBuilder();
    for (String key : keys) {
        String val = params.get(key);
        if (val != null && !val.isEmpty()) {
            if (sb.length() > 0) sb.append("&");
            sb.append(key).append("=").append(val);
        }
    }
    sb.append("&key=").append(merchantKey);
    // MD5
    MessageDigest md = MessageDigest.getInstance("MD5");
    byte[] digest = md.digest(sb.toString().getBytes("UTF-8"));
    StringBuilder hex = new StringBuilder();
    for (byte b : digest) hex.append(String.format("%02X", b));
    return hex.toString();
}

PHP

PHP<?php
function generateSign($params, $merchantKey) {
    // Exclude sign and lang
    unset($params['sign'], $params['lang']);
    // Sort by key
    ksort($params);
    // Concatenate
    $parts = [];
    foreach ($params as $k => $v) {
        if ($v !== '' && $v !== null) {
            $parts[] = "{$k}={$v}";
        }
    }
    $signStr = implode('&', $parts) . "&key={$merchantKey}";
    // MD5 and convert to uppercase
    return strtoupper(md5($signStr));
}

Node.js

JAVASCRIPTconst crypto = require("crypto");

function generateSign(params, merchantKey) {
    // Exclude sign and lang
    const { sign, lang, ...rest } = params;
    // Sort by key
    const sortedKeys = Object.keys(rest).sort();
    // Concatenate
    const signStr = sortedKeys
        .filter(k => rest[k])
        .map(k => `${k}=${rest[k]}`)
        .join("&") + `&key=${merchantKey}`;
    // MD5 and convert to uppercase
    return crypto.createHash("md5").update(signStr, "utf8").digest("hex").toUpperCase();
}
