Request Headers

Doc Version: V1.0

This page describes the HTTP request headers required when calling the Fintech Payment OpenAPI.

Standard Request Headers

When calling an endpoint using JSON POST, set the following request headers:

HeaderValueRequiredDescription
Content-Typeapplication/jsonRequiredRequest body is JSON
Acceptapplication/jsonRecommendedResponse body is JSON
Accept-Charsetutf-8RecommendedCharacter encoding is UTF-8

Hosted Mode Request Headers

Global Payments Online hosted mode uses an HTML form submission, so the request headers differ:

HeaderValueDescription
Content-Typeapplication/x-www-form-urlencodedForm submission format
ℹ️ Character Encoding

All requests and responses use UTF-8 encoding. Signature computation must also use UTF-8 to convert strings to byte streams.
