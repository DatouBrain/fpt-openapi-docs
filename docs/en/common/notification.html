Payment Notification

Doc Version: V1.0

After payment is completed, the platform asynchronously notifies the merchant of the payment result. This page describes the notification mechanism, retry strategy, and best practices.

Notification Mechanism

After payment is completed, the system sends the payment result to the notify_url provided when creating the order:

  • Notification Method: POST request, with a JSON data body
  • Notification URL: the notify_url parameter submitted in the create order endpoint
  • Notification content includes order no, transaction id, amount, payment status, etc., with a sign signature

Retry Strategy

If the platform does not receive the plain string success as the merchant's response, or no response is returned within 5 seconds, the platform considers the notification failed and will re-send the notification according to the following strategy:

Retry CountInterval
1st0 seconds (immediately)
2nd15 seconds
3rd15 seconds
4th30 seconds
5th180 seconds (3 minutes)
6th1800 seconds (30 minutes)
7-9th1800 seconds (30 minutes)
10th3600 seconds (1 hour)
⚠️ Note

The platform strives to maximize notification success rate but does not guarantee final delivery. Merchants are advised to also use active query (calling the /query endpoint) as a fallback.

Best Practices

Since backend notifications may be re-sent, the same notification may be delivered multiple times. Merchant systems must correctly handle duplicate notifications:

  1. Verify Signature: After receiving a notification, first verify the sign field to prevent forgery
  2. Verify Order: Verify the order_no and amount in the notification match your business system
  3. Idempotency: Check if the order was already processed; if so, return success directly
  4. Concurrency Control: Use data locks for concurrency control to avoid re-entry causing data corruption
  5. Respond Promptly: Return the plain string success immediately after processing (within 5 seconds) to avoid triggering retries

Notification Handling Code Example

PYTHON@app.route("/notify", methods=["POST"])
def handle_notify():
    data = request.get_json()

    # 1. Verify signature
    recv_sign = data.pop("sign", "")
    if generate_sign(data) != recv_sign:
        return "FAIL"

    # 2. Verify order
    order_no = data.get("order_no")
    amount = data.get("amount")
    order = get_order_from_db(order_no)
    if not order or float(order.amount) != float(amount):
        return "FAIL"

    # 3. Idempotency
    if order.status == "PAID":
        return "success"  # Already processed, return success directly

    # 4. Update order status (lock for concurrency)
    with db_lock:
        update_order_status(order_no, "PAID")

    # 5. Return success
    return "success"
