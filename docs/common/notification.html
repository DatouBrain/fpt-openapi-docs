支付通知說明

文檔版本：V1.0

支付完成後，平台會異步通知商戶支付結果。本頁面說明通知機制、重試策略及處理建議。

通知機制

支付完成後，系統會將支付結果通知到下單時填寫的 notify_url：

  • 通知方式：POST 請求，數據體為 JSON 格式
  • 通知 URL：下單接口中提交的 notify_url 參數
  • 通知內容包含訂單號、交易號、支付金額、支付狀態等信息，並附帶 sign 簽名

重試策略

如果平台收到商戶的應答不是純字符串 success，或超過 5 秒未返回，平台認為通知失敗，將按以下策略重新發起通知：

重試次數間隔時間
第 1 次0 秒（立即）
第 2 次15 秒
第 3 次15 秒
第 4 次30 秒
第 5 次180 秒（3 分鐘）
第 6 次1800 秒（30 分鐘）
第 7-9 次1800 秒（30 分鐘）
第 10 次3600 秒（1 小時）
⚠️ 注意

平台會盡可能提高通知的成功率，但不保證通知最終能成功。建議商戶同時使用主動查詢（調用 /query 接口）作為兜底方案。

處理建議

由於存在重新發送後台通知的情況，同樣的通知可能會多次發送給商戶系統。商戶系統必須能夠正確處理重複通知：

  1. 驗證簽名：收到通知後，先驗證 sign 字段是否正確，防止偽造
  2. 校驗訂單：校驗通知中的 order_no 和 amount 與自身業務系統的訂單和金額是否一致
  3. 冪等處理：檢查對應訂單是否已處理過，如已處理直接返回 success
  4. 併發控制：採用數據鎖進行併發控制，避免函數重入造成數據混亂
  5. 及時應答：處理完成後立即返回純字符串 success（5 秒內），避免觸發重試

通知處理代碼示例

PYTHON@app.route("/notify", methods=["POST"])
def handle_notify():
    data = request.get_json()

    # 1. 驗證簽名
    recv_sign = data.pop("sign", "")
    if generate_sign(data) != recv_sign:
        return "FAIL"

    # 2. 校驗訂單
    order_no = data.get("order_no")
    amount = data.get("amount")
    order = get_order_from_db(order_no)
    if not order or float(order.amount) != float(amount):
        return "FAIL"

    # 3. 冪等處理
    if order.status == "PAID":
        return "success"  # 已處理，直接返回成功

    # 4. 更新訂單狀態（加鎖防併發）
    with db_lock:
        update_order_status(order_no, "PAID")

    # 5. 返回成功
    return "success"
